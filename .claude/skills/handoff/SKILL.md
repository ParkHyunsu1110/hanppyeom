# handoff

이 skill은 작업을 마치기 전에 결과를 짧고 재사용 가능하게 정리하는 데 쓴다.
검증하지 않은 내용을 완료로 쓰지 않는다.
"끝났습니다"로 끝내지 않는다. **다음 액션 후보**를 반드시 남긴다.

## 포함할 내용

- 작업 목표와 실제 완료 범위
- 수정한 파일
- 생성/수정한 문서와 이유
- skill discovery 결과 — `skill-craft`로 만든 custom skill, `docs/skill-recommendations.md` 생성/생략 이유, 외부 raw copy 여부
- 실행한 검증과 결과
- 실행하지 못한 검증과 이유
- 남은 리스크 또는 사용자 결정이 필요한 항목
- **다음 액션 후보**(필수) — 사용자가 한 줄로 고를 수 있게 1~3개 제시

## 원칙

- 긴 배경 설명보다 다음 작업자가 바로 이어받을 사실을 우선한다.
- 추정과 확인된 사실을 분리한다.
- 실패한 검증을 숨기지 않는다.
- 새 custom skill이 필요해 보이면 반복 절차와 검증 기준이 명확한 경우에만 후보로 남긴다.
- 외부 skill은 참고 source로만 기록하고 raw copy하지 않았는지 분리해 쓴다.
- 다음 액션은 사용자가 결정 비용 없이 한 단어로 답할 수 있게 제시한다.

## 출력 형식

```text
## handoff

- 완료 범위: <요약>
- 변경 파일: <목록>
- 문서 처리: <생성/수정/생성 안 함과 이유>
- skill discovery: <skill-craft 결과, 채택/보류, 문서 생성 여부>
- 검증: <실행한 명령과 결과>
- 미검증/리스크: <항목>

## 다음 액션 후보
1. <한 줄 — 예: "/verify 다시 돌려서 stale 참조 재확인">
2. <한 줄 — 예: "skill-craft로 release-check skill 만들기">
3. <한 줄 — 예: "/handoff 종료, 새 세션으로 진행">

어느 것을 진행할까요?
```
