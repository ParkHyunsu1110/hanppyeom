# Phase 2: 성장기록 + 백분위 Design Review

> 상태: 진행 중 · 범위: 입력→저장→LMS 백분위→그래프 수직 슬라이스 · SSOT: `CLAUDE.md` 5·7·10장, `lib/models/`, `lib/services/growth/`

## 목표
- 부모가 아이의 키/체중/머리둘레를 기록하면, 또래 백분위 곡선 위에 측정점이 찍히고 현재 백분위 + 이전 대비 추세를 본다.
- "또래 평균"이 아니라 **백분위**(LMS 방법)로 보여준다.
- 기준표(2017 질병관리청/WHO)는 2027 개정 예정이므로 **교체 가능한 구조**로 둔다.

## 사용자 시나리오
- 주: 부모가 성장기록 화면에서 유형(키/체중/머리둘레) 선택 → 값 입력 → 저장 → 곡선 위 측정점 + 현재 백분위(예: "상위 38%") + 추세(↑/↓) 확인.
- 보조: 친척(RELATIVE)은 기록을 읽기만 한다(쓰기 불가). 측정값이 곡선 범위를 벗어나면 경고 없이 가장자리로 클램프해 표시.

## 입력 / 출력
- 입력: childId(=groupId), type(HEIGHT|WEIGHT|HEAD), value(cm/kg), date(측정일), recordedBy(uid).
- 출력/상태: `growthRecords/{id}`(groupId 비정규화) 생성. 백분위는 **저장하지 않고** 읽을 때 LMS로 계산.
- 계산 입력: 측정값 X + 해당 성별·개월수의 L, M, S(번들 참조표) → Z → 백분위.

## 제약
- 기술: Flutter + Firestore. 백분위 계산은 순수 Dart(오프라인). 참조표는 앱 번들(읽기 전용).
- 규칙: growthRecords는 해당 그룹 ACTIVE 멤버만 읽기, PARENT만 쓰기(멤버십 게이트, Phase 1과 동일 패턴).
- 데이터: LMS 표(성별·유형·개월수별 L,M,S)는 공공데이터포털 "영유아성장도표 LMS기준"에서 받아 번들. 이번 슬라이스는 계산기 + 데이터 인터페이스까지, **전체 표 번들은 후속 데이터 작업**.

## LMS 계산 (Box-Cox)
- `Z = ((X/M)^L − 1) / (L·S)` (L ≠ 0), `Z = ln(X/M) / S` (L = 0)
- `percentile = Φ(Z)` (표준정규 CDF, Abramowitz–Stegun 근사)
- 개월수 매칭: 측정일 − 생년월일 → 개월수, 표에서 (sex,type,ageMonths) 조회. 정확 개월 없으면 인접값 보간(이번 슬라이스는 최근접, 보간은 후속).

## 엣지케이스
- 참조표에 해당 (sex,type,age)가 없음 → 백분위 null 반환(점만 찍거나 "기준 없음" 표시).
- 값이 0/음수/비현실적 → 입력 검증에서 거부.
- 곡선 밖 극단값 → 백분위는 계산하되 0~100으로 자연 클램프(Φ가 처리).
- 개월수 0(신생아)·범위 밖 나이 → 표 범위 밖이면 null.

## 검증 기준
- 기능: X=M이면 백분위=50%; Z=+1→약84.1%, Z=−1→약15.9%; 단조성(값↑ → 백분위↑) 유닛 테스트.
- 기능: 기록 저장 후 그룹 멤버가 읽고, 친척은 쓰기 거부(규칙 테스트/에뮬레이터).
- 비기능: `flutter analyze`/`flutter test` 통과. 계산은 Firebase 불필요(순수 함수).

## Out of Scope (이번 슬라이스)
- 전체 LMS 표 번들·개월수 보간(데이터 작업으로 분리).
- 그래프 위젯 폴리시(축/곡선 음영 등)는 최소 구현 후 개선.
- 기준표 버전 토글 UI(구조만 교체 가능하게).

## 미해결 질문
- LMS 표 데이터 포맷(JSON asset vs 코드 상수): 우선 JSON asset + 로더로 가되, 확정은 데이터 번들 작업 시.
- 머리둘레(HEAD) 측정 표 범위(보통 0–35개월) — 표 확보 시 확정.
